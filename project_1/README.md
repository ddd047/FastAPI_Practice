# FastAPI Projects and Examples

This repository serves as a collection of learning examples and small projects utilizing the FastAPI web framework for building APIs with Python. It demonstrates basic routing, endpoint creation, and serving JSON responses.

## Getting Started

To run the applications in this repository, you'll need Python installed.

### Prerequisites

*   Python 3.7+
*   `pip` (Python package installer)

### Installation

1.  Clone this repository:
    ```bash
    git clone <repository-url>
    cd <repository-name>
    ```
2.  Install the necessary dependencies:
    ```bash
    pip install "fastapi[all]" uvicorn
    ```

## Usage

This repository contains multiple FastAPI applications that can be run independently.

### Running the Root `main.py` Application

To start the primary application located in the root directory:

```bash
uvicorn main:app --reload
```

Once running, you can access the application locally at `http://127.0.0.1:8000`.

*   **Root Endpoint**: Access `http://127.0.0.1:8000/` to receive `{"message": "HELLO WORLD !"}`.
*   **About Endpoint**: Access `http://127.0.0.1:8000/about` to receive `{"message": "I am Himanshu. And I am Learning FastAPI"}`.
*   **API Documentation**: View the auto-generated interactive API documentation at `http://127.0.0.1:8000/docs`.

### Running the `my-new-folder/main.py` Application

To start the application located within the `my-new-folder` directory, you can either navigate into that directory first or specify the module path from the root:

```bash
# Option 1: Navigate and run
cd my-new-folder
uvicorn main:app --reload

# Option 2: Run from root directory
# uvicorn my-new-folder.main:app --reload
```

This application also typically runs on `http://127.0.0.1:8000` (if no other application is occupying the port). It provides identical endpoints:

*   **Root Endpoint**: `http://127.0.0.1:8000/` will return `{"message": "HELLO WORLD !"}`.
*   **About Endpoint**: `http://127.0.0.1:8000/about` will return `{"message": "I am Himanshu. And I am Learning FastAPI"}`.
*   **API Documentation**: Access the auto-generated interactive API documentation at `http://127.0.0.1:8000/docs`.

## Project Structure and Files

Here's an overview of the key files and directories in this repository:

*   **`main.py`**:
    *   This is a FastAPI application script located in the repository's root directory.
    *   It defines two GET endpoints: `/` (a basic "Hello World" message) and `/about` (a personal introduction message).
    *   It demonstrates the fundamental setup for a FastAPI application, including importing `FastAPI` and defining routes using decorators.
    *   Includes inline comments providing instructions on how to run the server and access the API documentation.
    *   It imports `pydantic.BaseModel`, suggesting potential future use for data validation or request/response models.

*   **`my-new-folder/`**:
    *   A directory containing another independent FastAPI application.
    *   **`my-new-folder/main.py`**:
        *   Similar to the root `main.py`, this script implements a FastAPI application with `/` and `/about` endpoints.
        *   It provides identical functionality and messages to the root `main.py`, showcasing a separate instance of a basic FastAPI service.
        *   It does not explicitly import `pydantic.BaseModel` in this version.

*   **`file.pdf`**:
    *   A PDF document included in the repository. Its specific content is not detailed.

*   **`__pycache__/` directories and `.pyc` files**:
    *   These directories and `.pyc` files are automatically generated by Python to store compiled bytecode from Python scripts. They help optimize script execution speed but are typically ignored in version control and project documentation as they are not source code.
